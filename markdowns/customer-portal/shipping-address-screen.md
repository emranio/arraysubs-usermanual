# User Guide
The **Shipping Address** screen appears on the Subscription Details page for subscriptions that actually require shipping.

Customers usually reach it from:

- **My Account → Subscriptions → View subscription → Shipping Address**

This section helps customers update where future renewal shipments should go.

## When the Shipping Address section appears
The section does not appear for every subscription.

Customers only see it when the subscription is set up as a shipped subscription.

That means digital-only subscriptions or subscriptions that do not use shipping will not show this screen at all.

## What the screen shows
When shipping applies, the screen can show:

- the current shipping address on file
- a note about how shipping is charged
- an **Update Shipping Address** button when updates are allowed
- a cutoff notice when updates are temporarily blocked

## Shipping-type note
The screen can explain whether shipping is:

- charged on each renewal order
- charged only on the initial order

This matters because customers often assume all subscription shipping behaves the same way.

## The saved address display
If an address exists, the portal displays it in a normal address format.

Customers may see details such as:

- first and last name
- company
- street address
- city, state, and postcode
- country

If no address is stored yet, the portal instead tells the customer that there is no shipping address on file.

## Update Shipping Address button
When updates are allowed, the section shows:

- **Update Shipping Address**

Clicking that button opens a modal dialog where the customer can edit the fields for future renewal shipments.

## The update modal
The address modal includes the main fields customers expect:

- First Name
- Last Name
- Company
- Street Address
- Apartment, suite, etc.
- City
- State / Province
- Postcode / ZIP
- Country
- Phone

The customer saves the changes with **Save Address**.

## Required fields
Some fields are required before the form can be saved.

Customers should expect required validation for core address details such as:

- first name
- last name
- street address
- city
- postcode
- country

If a required field is missing, the portal shows an error instead of saving incomplete data.

## Address cutoff rules
One of the most important parts of this screen is the shipping update cutoff.

### What the cutoff means
Customers cannot always change the shipping address right before renewal.

If the next payment date is too close, the portal can block changes and show a notice explaining that shipping address changes are no longer allowed within the cutoff window.

### Why this matters
This protects renewal processing and avoids situations where:

- a renewal order is already being prepared
- fulfillment timing would become unreliable
- the customer expects an address change that happens too late to apply

## What customers should do if the cutoff has passed
If the portal shows a cutoff message instead of the update button, the customer should understand that:

- the next shipment may still go to the current saved address
- the address can often be updated again after the blocked window passes
- support may need to advise whether the next order is already committed to fulfillment

# Use Case
A customer is moving next week and wants future subscription shipments sent to a new address. They open the subscription detail page, review the Shipping Address section, click **Update Shipping Address**, and save the new details before the renewal cutoff window begins.

# FAQ
### Why do some subscriptions not show a Shipping Address section?
Because only subscriptions that require shipping display this section.

### Can customers update shipping at any time?
No. Address updates can be blocked close to the next payment date.

### What happens if there is no saved shipping address yet?
The portal shows that no shipping address is on file, and the customer can usually add one if updates are allowed.

### Does updating this screen affect future renewals or past orders?
It is meant for future renewal shipments, not for rewriting past completed orders.
