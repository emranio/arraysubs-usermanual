# User Guide
This topic explains how store admins and support teams manage subscriptions inside the ArraySubs admin experience.

Use this section when you need to work with live subscription records after products are already configured and customers have started purchasing.

## Where to go in wp-admin
The main entry point is:

- **WordPress Admin → ArraySubs → Subscriptions**

From there, ArraySubs gives admins a working flow for:

- browsing and filtering subscriptions
- opening full subscription detail pages
- reviewing notes, payment history, and linked orders
- editing operational fields such as dates, invoice email, and addresses
- creating subscriptions manually for customers
- cancelling subscriptions and undoing scheduled cancellation
- jumping into WooCommerce orders for refund and order handling

## What is part of core and what is extra
Most day-to-day subscription management is part of the core ArraySubs experience.

Some detail-page cards can expand when premium modules are active. For example:

- **Pro** payment-gateway details can appear for automatic payments
- **Pro** subscription shipping details can appear when recurring shipping features are enabled

That means the same admin screens can show more information without becoming a completely separate workflow.

A support team needs one place to learn how ArraySubs works in the back office: find the customer, inspect the subscription, review renewal orders, add notes, cancel correctly, and update the operational details without guessing which screen does what. This topic is that playbook.

# FAQ
### Is this topic only about the subscription list?
No. It covers the full admin handling flow from the list page to the detail page, edit form, manual creation form, and order-related follow-up work.

### Is this separate from WooCommerce orders?
No. ArraySubs manages the subscription record, but the related orders still matter. This topic shows how the subscription detail page surfaces those linked WooCommerce orders.

### Does this topic include premium-only details?
Yes. Core behavior is documented first, and premium-only detail-page additions are clearly called out where relevant.
