# User Guide

## Overview

This section covers the two main settings screens inside **ArraySubs → Settings**: General Settings and Toolkit Settings.

General Settings controls how subscriptions behave across your store — billing rules, checkout options, renewal timing, grace periods, customer self-service, and cancellation policies. Toolkit Settings manages admin experience, login security, and session controls.

Other features such as Retention Flow, Plan Switching, Skip & Pause, Refunds, Payment Gateways, Checkout Builder, Feature Manager, and Store Credit each have their own dedicated settings screens. Those are documented inside their own manual topics rather than here.

## When to Use This

Use this section when you are:

- configuring ArraySubs for the first time after installation
- reviewing subscription cart and checkout rules
- adjusting renewal sync, grace periods, or email reminders
- enabling or disabling customer self-service actions
- setting up admin bar, login page, and session security controls
- training support staff on what each setting controls

## Topics

1. [General Settings](./general-settings.md) — subscription behavior, checkout, billing, trials, renewals, grace periods, customer portal, self-service actions, and cancellation timing
2. [Toolkit Settings](./toolkit-settings.md) — admin bar visibility, wp-admin access, login page control, admin impersonation, and multi-login prevention

## Prerequisites

- ArraySubs installed and active
- Admin or Shop Manager access to **ArraySubs → Settings**
- ArraySubsPro installed for premium-only settings marked with **Pro**

# FAQ

### Where do I find settings for specific features like Retention Flow or Store Credit?
Those features have their own settings screens and their own manual topics. This section only covers the General Settings and Toolkit Settings pages.

### Do I need Pro to use General Settings?
No. General Settings is part of the core plugin. Some fields such as automatic-payment customer controls require Pro modules to be active before they become relevant.

### What happens if I change a setting after subscriptions already exist?
Most settings affect future subscription actions only. Existing subscriptions keep their billing schedule, pricing, and status unless you edit them directly. The settings reference in each guide explains what changes immediately and what only applies going forward.
